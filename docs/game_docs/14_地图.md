## 一、地图总览

### 1.1 地域制压系统（替代节点跳跃）

游戏地图由 **55 个地域（Territory）** 组成，每个地域是一个有面积的区域（非抽象节点）。
玩家通过 **派遣军团进攻相邻地域** 来扩张版图，而非在节点间移动棋子。

核心区别（vs 旧版节点系统）：
| 旧版 | 新版 |
|------|------|
| 棋子在节点间移动 | 军团从己方领地出击到相邻领地 |
| 骰子决定移动距离 | AP直接选择目标地域 |
| 单一玩家位置 | 多军团同时驻扎不同领地 |
| 到达节点触发事件 | 占领地域后触发事件/生产 |

### 1.2 地图规模
- **总地域数**: 55
- **地域连接**: Delaunay三角剖分 + 手动调整，保证每个地域2-5个邻接
- **概念尺寸**: 1920×1080逻辑空间（2.5D投影后视觉尺寸）
- **地域最小间距**: 120px（中心点距离）

## 五、军团与行动系统（v0.9.2 新增）

### 5.1 军团系统

玩家和AI通过 **军团（Army）** 控制地域。每个地域可以驻扎军团或仅有驻军。

| 概念 | 说明 |
|------|------|
| 驻军（Garrison） | 地域的固定防御兵力，不可移动，每回合自动恢复 |
| 军团（Army） | 可移动的军事力量，由部队编制+英雄组成 |
| 部队编制 | 军团内的部队组合（参考11_兵种数据包） |
| 英雄配属 | 最多2名英雄配属到1支军团 |

**军团限制:**
- 玩家最多同时拥有 **3支军团**（可通过建筑升级到5支）
- 每支军团最多 **5个部队编制**（3前排+2后排）
- 军团必须驻扎在己方地域
- 同一地域最多驻扎 **1支军团**

### 5.2 回合行动流程

每回合玩家拥有 **AP（行动点）**:
- 基础AP: 2
- 每5个己方地域: +1 AP
- 上限: 6 AP

**行动消耗:**
| 行动 | AP消耗 | 说明 |
|------|--------|------|
| 出击（Attack） | 1 AP | 选择己方军团 → 选择相邻敌方/中立地域 → 战斗 |
| 防御部署（Deploy） | 1 AP | 将军团移动到相邻己方地域 |
| 内政（Domestic） | 1 AP | 招募/升级/建造（在军团所在地域） |
| 外交（Diplomacy） | 1 AP | 与中立势力交涉 |
| 探索（Explore） | 1 AP | 探索相邻未知/事件地域 |
| 强行军（Forced March） | 2 AP | 军团移动2格，部队损耗10%兵力 |
| 结束回合 | 0 AP | 进入AI回合 |

**自由行动（不消耗AP）:**
- 英雄管理（装备/赠礼/技能查看）
- 科技训练（查看进度/选择研究）
- 存档/读档
- 查看地域信息

### 5.3 出击流程

```
1. 选择己方军团（从军团列表或点击地图上的己方军团）
2. 显示该军团可攻击的相邻地域（高亮标记）
3. 点击目标地域
4. 进入战斗结算
5. 胜利 → 军团占领该地域（驻军替换为胜利军团）
6. 失败 → 军团撤回出发地域（损失部队）
```

### 5.4 防御机制

当己方地域被进攻时:
- 如果有驻扎军团: 军团+驻军共同防御
- 如果无军团: 仅驻军防御（战力较低）
- 驻军每回合自动恢复: +1兵（城墙地形+2兵）
- 驻军上限: 地域等级 × 5

## 六、2.5D可视化（v0.9.2 新增）

### 6.1 视角设定

| 参数 | 值 |
|------|-----|
| 投影方式 | 透视投影 (FOV 40°) |
| 摄像机俯角 | 55°（近似等距投影效果） |
| 摄像机高度 | Y=25 |
| 地图朝向 | 菱形布局（45°旋转） |
| 缩放范围 | 0.5x ~ 2.0x |
| 平移范围 | 地图边界±5单位 |

### 6.2 地域可视化

每个地域在3D空间中表现为:
1. **地面区域**: 略微凸起的六边形/不规则多边形地面
2. **地形层**: 根据地形类型添加装饰（树木/岩石/水面）
3. **建筑层**: 地域上的建筑（城墙/村庄/塔楼等）
4. **旗帜**: 标识所属阵营的旗帜（颜色编码）
5. **军团标记**: 驻扎军团的视觉标记（带兵力数字）

### 6.4 摄像机控制

| 操作 | 输入 |
|------|------|
| 平移 | WASD / 鼠标中键拖拽 / 边缘滚动 |
| 缩放 | 滚轮 / +/- 键 |
| 聚焦地域 | 双击地域 / 小地图点击 |

### 6.5 交互流程

```
1. 鼠标悬停地域 → 高亮边框 + 右侧面板显示地域信息
2. 左键点击己方地域 → 选中（显示军团信息/可用行动）
3. 左键点击敌方地域 → 若有选中军团且相邻 → 确认出击
4. 右键 → 取消选择
5. 选中军团后，相邻可达地域显示不同高亮：
   - 绿色: 己方地域（可部署移动）
   - 红色: 敌方地域（可进攻）
   - 黄色: 中立地域（可探索/外交）
```

## 七、地图生成算法（v0.9.2 更新）

```
1. 在1920×1080逻辑空间中放置8个要塞节点（最大间距，按区域固定位置）
2. 放置6个中立据点（距要塞最少200px）
3. Poisson Disk采样填充剩余节点至55个（最小间距120px）
4. Delaunay三角剖分建立邻接关系
5. 移除过长的边（>300px），确保每节点保留2-5个邻接
6. 按区域分配阵营所有权
7. 分配地域类型（含特殊据点: 交易站/瞭望塔/遗迹/港口/关隘）
8. 按区域权重表分配地形
9. 识别自然关隘（邻接度≤3 + 防御性地形）
10. 应用预设命名据点
11. 为每个地域生成Voronoi区域轮廓（视觉用）
```

## 八、迷雾系统

- 未探索地域: 灰色蒙版，无法查看详情
- 己方地域视野: 由地形决定（平原2格/森林1格/山地3格）
- 瞭望塔额外+2视野
- 占领过的地域永久可见（但驻军信息仅实时可见）
- 相邻己方地域的敌方驻军信息可见

---

## 九、补给线系统

- 军团所在地域到最近己方核心要塞的最短路径（仅经过己方地域）
- 路径长度 ≤ 5: 正常
- 路径长度 > 5: 每回合该军团-2兵
- 无法连通己方核心: 每回合-5兵（孤军）
- 补给线可以被敌方切断（占领路径上的地域）
