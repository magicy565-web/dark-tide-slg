# 08 存档加载机制

## 一、存档系统设计

### 存档策略

- **自动存档**: 每回合结束时自动保存1个槽位
- **手动存档**: 策略阶段(Phase 4)可随时保存，最多5个槽位
- **快速存档**: F5快速保存, F9快速加载
- **存档位置**: `user://saves/`

## 二、保存流程

```
1. 暂停游戏逻辑
2. 序列化GameManager全部状态→Dictionary
3. 序列化地图状态（节点/边/所有权）
4. 序列化军队（位置/编制/兵数）
5. 序列化英雄（俘获/招募/好感/装备）
6. 序列化事件（已触发/进行中buff/任务进度）
7. 序列化标记（剧情flag/征服记录）
8. 写入JSON文件 + 截图缩略图
9. 恢复游戏逻辑
```

---

## 三、加载流程

```
1. 读取JSON文件
2. 验证版本兼容性
3. 重置GameManager状态
4. 恢复地图（用保存的seed重新生成骨架，覆盖动态数据）
5. 恢复资源/秩序值/威胁值/机制值
6. 恢复军队编制
7. 恢复英雄状态
8. 恢复事件/buff/任务进度
9. 恢复剧情标记
10. 刷新UI
```

---

## 四、版本兼容

- 每个存档记录版本号
- 加载时检查版本，若不匹配：
  - **小版本**(1.0→1.1): 自动迁移，补充缺失字段默认值
  - **大版本**(1.x→2.x): 提示不兼容，拒绝加载

---

## 五、防作弊

- 存档文件加校验和(CRC32)
- 加载时验证，校验失败提示"存档可能已损坏"
- 不阻止加载（单机游戏不需要严格防作弊）
