# 17_教程关卡设计

> 《暗潮 Dark Tide》教程关卡系统设计文档

## 一、设计目标
为玩家提供一个平滑的上手体验，通过一个独立的、受控的“教程关卡”场景，引导玩家逐步了解游戏的核心机制：内政、行动、任务、战斗、事件、地域压制、交易、外交。

## 二、教程关卡架构
教程关卡将作为一个独立的场景（`tutorial_level.tscn`）和管理器（`tutorial_level_manager.gd`）存在，不干扰主游戏的随机生成和复杂逻辑。

### 1. 独立地图与数据
- **固定地图**：使用一个只有5-6个节点的微型地图。
  - 节点0：玩家主城（村庄）
  - 节点1：空地（用于移动和内政演示）
  - 节点2：弱小匪巢（用于初次战斗演示）
  - 节点3：中立资源站（用于任务和事件演示）
  - 节点4：敌方要塞（用于地域压制和外交演示）
- **受控环境**：禁用随机事件、AI主动攻击和复杂的经济衰减，直到教程特定阶段。

### 2. 引导系统升级
- 扩展现有的 `TutorialManager`，增加对特定UI元素的强制高亮和点击拦截（Masking）。
- 增加“目标检测”机制，只有玩家完成特定操作（如“招募1队士兵”），才能进入下一步。

## 三、教程流程规划（8个阶段）

### 阶段1：基础认知与移动（回合1）
- **目标**：了解界面、资源、如何移动军队。
- **流程**：
  1. 欢迎对话框，介绍背景。
  2. 高亮顶部资源栏，解释金币、粮食、行动力（AP）。
  3. 高亮玩家主城和军队。
  4. 引导玩家点击军队，下达移动指令到节点1。
  5. 引导玩家点击“结束回合”。

### 阶段2：内政与招募（回合2）
- **目标**：学习如何发展经济和扩充军力。
- **流程**：
  1. 解释回合收入和粮食消耗。
  2. 引导玩家在节点1执行“内政-建造”操作（如建造农田）。
  3. 引导玩家在主城执行“招募”操作，补充兵力。
  4. 结束回合。

### 阶段3：初次战斗（回合3）
- **目标**：理解战斗系统、排兵布阵。
- **流程**：
  1. 引导玩家移动军队攻击节点2（匪巢）。
  2. 进入战斗准备界面，强制引导玩家将近战放前排，远程放后排。
  3. 解释战斗预测和兵种克制。
  4. 确认开战，观看战斗结果。
  5. 战斗胜利后，解释战利品和经验值。

### 阶段4：任务与事件（回合4）
- **目标**：了解中立势力任务和随机事件。
- **流程**：
  1. 军队移动到节点3（中立资源站）。
  2. 触发预设的“初遇中立势力”事件。
  3. 引导玩家打开任务日志（Quest Journal），查看任务要求（如交纳少量金币）。
  4. 引导玩家完成任务，获得奖励（如临时增益或资源）。

### 阶段5：地域压制与城防（回合5）
- **目标**：学习攻城战和领地扩张。
- **流程**：
  1. 引导玩家攻击节点4（敌方要塞）。
  2. 解释“城防”概念：需要先消耗城防才能对驻军造成全额伤害。
  3. 战斗胜利后，弹出“地域压制”选择（占领、掠夺、摧毁）。
  4. 引导玩家选择“占领”，解释秩序值和叛乱风险。

### 阶段6：交易与经济调控（回合6）
- **目标**：学习如何处理资源不平衡。
- **流程**：
  1. 教程强制扣除玩家大量粮食，制造“缺粮危机”。
  2. 引导玩家打开“省份信息面板”或“外交面板”。
  3. 引导玩家使用“交易”功能，用金币购买粮食。
  4. 解释奴隶市场（如果使用海盗/暗精灵阵营）或黑市。

### 阶段7：外交与收编（回合7）
- **目标**：了解外交选项和不战而屈人之兵。
- **流程**：
  1. 节点4后方出现一个新的强大中立势力节点5。
  2. 引导玩家打开外交面板。
  3. 解释威望、关系值。
  4. 引导玩家执行“赠送礼物”或“签订互不侵犯条约”。
  5. 解释如何通过外交“收编”势力。

### 阶段8：自由探索与结语
- **目标**：总结教程，鼓励玩家开始正式游戏。
- **流程**：
  1. 恭喜玩家完成所有基础训练。
  2. 解锁所有UI限制。
  3. 提供一个按钮：“退出教程并开始新游戏”。

## 四、技术实现路径

1. **创建 `tutorial_level.gd`**：作为教程关卡的控制器，继承自 `Node`，挂载在 `main.tscn` 或作为独立场景。
2. **扩展 `tutorial_manager.gd`**：
   - 增加 `set_mask_enabled(bool)`：阻止玩家点击非高亮区域。
   - 增加 `wait_for_action(action_name)`：等待特定事件总线信号。
3. **配置固定地图数据**：在 `GameManager` 或 `MapGenerator` 中增加 `generate_tutorial_map()` 方法。
4. **注入预设事件**：使用 `EventBus` 手动触发教程所需的特定事件，绕过随机数生成器。
