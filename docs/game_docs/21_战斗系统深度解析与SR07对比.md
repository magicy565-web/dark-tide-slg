# 《暗潮》战斗系统深度解析与《战国兰斯07》对比研究

本文档对《暗潮》当前的战斗系统进行了全面梳理，并与经典的《战国兰斯07》（Sengoku Rance）战斗机制进行了深度对比分析，指出了当前设计中的缺口以及未来的改进方向。

## 一、 核心伤害公式对比

《战国兰斯07》的战斗系统以“兵数”为绝对核心，通过非线性缩放和多层乘区计算，构建了易于上手但深度极高的战术体系。

### 1.1 战国兰斯07 的伤害模型

在《战国兰斯07》中，物理伤害的基本计算公式为：
**伤害 = (攻击力 - 防御力) × 兵数修正 × 技能修正 × 其他修正** [1]

其中，攻击力和防御力的计算融合了武将属性与兵种基础值：
- **攻击力** = 武将攻 × 10 × 攻付与率 + 兵种攻
- **防御力** = 武将防 × 8 × 防付与率 - 兵种防

**核心设计点：兵数非线性修正**
为了防止大兵团在游戏后期形成绝对碾压，战国兰斯对兵数引入了分段非线性修正 [1]：
- 1-99兵：全部按100兵计算（保护残血部队）
- 100-499兵：1:1真实计算
- 500-1999兵：500 + (兵数-500)/2
- 2000-9999兵：1250 + (兵数-2000)/4

此外，游戏严格规定了**最终伤害上限不能超过攻击方的当前兵数**，这使得“以少胜多”成为可能，玩家必须依靠技能、付与（Buff）和兵种克制来弥补兵力劣势。

### 1.2 《暗潮》的伤害模型

《暗潮》目前的伤害公式定义在 `combat_resolver.gd` 中：
**基础伤害 = 兵数 × max(1.0, 攻击力 - 防御力) / 10.0**

其中，攻击力与防御力的计算方式为：
- **攻击力 (atk)** = 兵种基础攻 + 英雄ATK + 阵型/地形/被动加成
- **防御力 (def)** = 兵种基础防 + 英雄DEF + 阵型/地形/被动加成

**对比分析与设计缺口：**
1. **线性兵数缩放**：《暗潮》的伤害与兵数呈绝对线性关系。这意味着1000兵的部队造成的伤害严格是100兵的10倍。在游戏后期，这极易导致数值崩坏和滚雪球效应。
2. **缺乏伤害上限限制**：《暗潮》没有限制单次伤害上限，高攻英雄带领少量兵力也能打出核弹级伤害，违背了SLG“兵力是基础”的直觉。
3. **攻防收益非对称**：在战国兰斯中，武将防的乘数是8，武将攻是10，且兵种防是减项。而在《暗潮》中，攻防完全对等，导致高防部队容易将伤害降至极低（虽然有最低1点伤害的保底）。

## 二、 战斗流程与战术维度对比

### 2.1 行动顺序与速度机制

在《战国兰斯07》中，行动顺序由“兵种速度 + 武将速参数”决定，但采用了反直觉的“速度悖论”设计：武将速参数越高，附加的值越小（速0=+8，速8=-8） [1]。此外，每个技能都有独立的“等待时间”（如音速手里剑等待2，火绳枪射击等待30），形成了一条动态的时间轴。

《暗潮》目前采用的是基于SPD（速度）的绝对排序：
- 每回合开始时，所有单位按 `SPD + randf()` 降序排列。
- 缺乏技能维度的速度修正，所有攻击（无论是普攻还是技能）都消耗相同的行动轮次。

### 2.2 状态增益（付与/Buff）系统

战国兰斯的“付与”系统极具策略性：
- 增益效果只看“有无”，不看概率。一旦附上，伤害直接×1.3或被伤×0.7。
- 军师的“知”属性决定了开局能施放多少个付与。
- 阵型位置（右上、右下、左上、左下）决定了付与的优先级。

《暗潮》的Buff系统更为传统，采用持续回合数和数值叠加机制。虽然有 `CommanderIntervention` 系统允许玩家在战斗中消耗CP（Command Points）施放战术指令，但在常规战斗的自动Buff流转上，缺乏战国兰斯那种“军师运筹帷幄”的阶段感。

### 2.3 胜负判定与战果指示条

《战国兰斯07》采用了经典的“战果指示条”拔河机制。12回合结束时，如果双方均未全灭，则根据战果条偏向决定胜负。这使得“铁壁防守流”成为一种极具价值的战术。

《暗潮》在近期的更新中（v4.7）已成功引入了类似机制：
- 引入了0-100的 `battle_rating`，初始为50。
- 每次造成伤害时，根据伤害量和目标最大兵力拨动战果条。
- 8回合超时后，>50进攻方胜，≤50防守方胜。
这一改进极大提升了战斗的策略深度，填补了之前的硬判定缺陷。

## 三、 改进建议与未来方向

基于上述深度对比，建议《暗潮》在未来的战斗模块更新中重点关注以下三个方向：

### 3.1 引入兵数非线性衰减机制

为了保证游戏后期的战斗体验，建议在 `combat_resolver.gd` 的伤害公式中引入类似战国兰斯的兵数修正曲线。
例如，当部队兵数超过500时，超出的部分在计算伤害时只提供50%的权重；超过2000时，超出部分只提供25%的权重。同时，强制设定单次攻击的基础伤害上限不能超过攻击方的当前兵数。

### 3.2 深化技能时间轴（Action Delay）

打破当前“每回合每人动一次”的僵化模式。可以为不同兵种和技能引入“行动后摇（Action Delay）”机制。
例如，火枪手（Pirate Archer）射击后增加大量Delay，导致其在接下来的两回合内无法行动；而刺客或轻骑兵普攻后Delay极小，一回合可能行动两次。这将极大丰富战术组合。

### 3.3 重构军师与Buff机制

参考战国兰斯的“付与”机制，强化INT（智力）属性在战斗中的战略地位。
当前INT主要用于计算Mana上限。建议引入“战前布阵阶段”，高INT的英雄可以在开战前自动为友军施加不可驱散的强力“战术付与”（如绝对防御一次、首击必暴击等），将智将的价值从单纯的法术伤害中解放出来。
