## 二、角色美术

> 角色完整设定参见 → `02_角色设定.md`

### 2.1 立绘规格

| 资源类型 | 数量/角色 | 分辨率 | 说明 |
|----------|-----------|--------|------|
| 基础立绘 | 1张 | 2048×3072 | 全身/半身立绘，用于对话和信息面板 |
| 表情差分 | 3-5张 | 面部区域 | 喜/怒/哀/惊/羞，替换面部图层 |
| 战斗小头像 | 1张 | 256×256 | 战斗界面单位标识 |
| 事件CG | 3张 | 1920×1080 | 对应3段调教/好感事件 |

### 2.2 阵营美术风格

**邪恶方角色（兽人/海盗/暗精灵）:**
- 哥特暗黑美学
- 暗色系盔甲、尖锐线条、冷色调
- 装饰元素: 骨饰、锁链、暗纹
- 皮肤色调偏冷（暗精灵为灰紫色肌肤）

**光明方角色（人类/精灵/法师）:**
- 日式幻想美型
- 明亮色调、柔和线条
- 人类方: 华丽铠甲、绸缎、金属光泽
- 精灵方: 自然元素、藤蔓纹饰、流线型
- 法师方: 法袍、宝石、魔法光效

**中立方角色:**
- 风格多样，依据角色据点环境
- 整体偏向实用主义设计（冒险者/商人/隐士风格）

### 2.3 角色命名与文化风格

全部角色采用日式命名（凛、雪乃、红叶、冰华、翠玲、月华等），角色造型在暗黑奇幻基底上融入日式审美元素。

### 3.2 整体色调原则

- **玩家方（邪恶）**: 以暗色为基调，红/紫作为主色
- **敌方（光明）**: 以明亮色为基调，金/白/绿作为主色
- **战斗界面**: 背景偏暗以突出角色立绘
- **地图界面**: 以羊皮纸色为底，阵营色区分领地

## 四、UI美术

> UI功能设计详见 → `09_UI.md`

### 4.1 整体布局参考

参考《战国兰斯》系列UI布局:

```
┌────────────────────────────────────────────┐
│  [资源栏: 金币/法力/兵力/威胁/威望/回合]    │  ← 顶部横条
├──────────────────────┬─────────────────────┤
│                      │                     │
│                      │    信息面板          │
│    地图区域          │    (角色/据点)       │  ← 主体区域
│                      │                     │
│                      │                     │
├──────────────────────┴─────────────────────┤
│  [行动栏: 出击/内政/外交/调教/结束回合]     │  ← 底部行动栏
└────────────────────────────────────────────┘
```

### 4.2 UI元素规范

| 元素 | 规范 |
|------|------|
| **字体** | 日式风格衬线字体（推荐 Noto Serif CJK），标题加粗 |
| **按钮** | 带纹理的方形按钮，深色底 + 金色边框，悬停时微亮 |
| **对话框** | 半透明暗色底板（透明度70-80%），角色立绘置于左侧 |
| **资源栏** | 顶部横条排列，每项为: 图标(32×32) + 数字 |
| **提示框** | 暗色底板 + 金色细边框，文字白色 |
| **选项按钮** | 横向排列，选中状态金色高亮 |

### 4.3 字体规范

| 用途 | 字体 | 字号 | 颜色 |
|------|------|------|------|
| 标题 | Noto Serif CJK Bold | 28-36px | 金色 #DAA520 |
| 正文 | Noto Serif CJK Regular | 18-22px | 白色 #F0F0F0 |
| 数值 | 等宽字体 | 20-24px | 对应阵营色 |
| 伤害数字 | 粗体 + 描边 | 36-48px | 红色(伤害)/绿色(治疗) |

## 五、战斗界面

### 5.1 布局设计

采用战国兰斯式 **"左右对阵立绘演出"** 模式:

```
┌──────────────────────────────────────────────┐
│                  [战斗信息栏]                  │
├──────────────────────┬───────────────────────┤
│    【玩家方·左侧】   │    【敌方·右侧】       │
│                      │                       │
│   [后排2] ○  ○      │      ○  ○ [后排2]     │
│   [前排3] ○  ○  ○   │   ○  ○  ○ [前排3]     │
│                      │                       │
├──────────────────────┴───────────────────────┤
│  [技能栏] [行动选择] [自动/快进]              │
└──────────────────────────────────────────────┘
```

- 左方为玩家军队，右方为敌方军队
- 每侧最多 **5个单位槽**（前排3 + 后排2）
- 布阵结构: **前排3 + 后排2**

### 5.2 单位表现

每个战斗单位的视觉组成:

| 元素 | 说明 |
|------|------|
| 指挥官半身立绘 | 缩小版立绘，用于识别角色 |
| 兵种小图标 | 左下角标识兵种类型（武士/弓兵/術師等） |
| 兵数显示 | 数字直接显示当前兵力 |
| HP条 | 兵数下方横条，绿→黄→红渐变 |
| 状态图标 | 右上角小图标显示buff/debuff |

### 5.3 战斗动画

| 动画类型 | 表现方式 |
|----------|----------|
| 普通攻击 | 攻击方立绘**前冲 + 闪烁** → 被攻击方**抖动** → 伤害数字弹出 |
| 技能攻击 | **全屏特效**（闪光/粒子）+ **技能名文字**居中显示 → 伤害数字弹出 |
| 治疗/辅助 | 绿色光效 + 数字向上飘动 |
| 击败 | 立绘**灰化 + 淡出** |
| 暴击 | 画面震动 + 加大伤害数字 + 特殊音效 |

## 六、地图美术

### 6.1 地图类型

采用战国兰斯式 **领地地图（节点型连接图）**:

- 不是传统的网格地图，而是以 **据点节点 + 连接路径** 构成
- 每个节点代表一个可攻略的据点或城池
- 连接线以道路样式绘制，表示进军路线

### 6.2 节点设计

| 元素 | 设计 |
|------|------|
| 节点形状 | 圆形或六角形图标 |
| 阵营标识 | 节点边框/填充色对应阵营配色（参见第三章） |
| 据点名称 | 节点下方文字标注 |
| 据点等级 | 节点大小区分（小/中/大 = 村落/城镇/要塞） |

### 6.3 地形与状态表现

| 地形/状态 | 视觉表现 |
|-----------|----------|
| 平原 | 绿色底纹 |
| 山地 | 棕色底纹 + 山峰图案 |
| 森林 | 深绿底纹 + 树木图案 |
| 沙漠 | 土黄底纹 + 沙丘图案 |
| 海域 | 蓝色底纹 + 波浪图案 |
| 未探索 | 灰色蒙版覆盖（战争迷雾） |
| 己方据点 | 明亮高光边框 |
| 可攻击据点 | 红色脉冲边框（呼吸灯效果） |
| 盟友据点 | 蓝色稳定边框 |

### 6.4 地图交互

- 点击节点: 弹出据点信息面板（驻军/建筑/资源）
- 拖拽: 地图平移
- 滚轮: 缩放
- 长按可攻击节点: 进入出击准备界面

> 地图详细数据参见 → `14_地图.md`
